# dbskill 新手入门

你可能是看到有人用 dbskill 推进真实任务，想知道自己该怎么开始。

dbskill 由自媒体博主 dontbesilent 创建，全网可搜 `@dontbesilent 聊赚钱`。dontbesilent 累计发布 16152 条推文，dbskill 将其中经过筛选的内容整理为 4176 个知识原子和 29 个 AI Skills，用来处理问题、材料、目标、选择和持续卡点。

你不需要记住工具名称，也不需要先把问题说得很专业。先把真实处境交出来，dbskill 会帮你找到切入点。

## 第一次怎么用

第一次使用，直接输入：

```text
/dbs 新手入门
```

dbskill 会说明你可以交付什么、系统怎样处理、可能得到什么结果。你提交真实任务后，它会选择合适的 Skill；每次只决定当前一步，后续方向由实际结果决定。

你也可以输入 `/dbs`，然后提交任何真实任务：

```text
我有一个问题还没想清楚：……
这段内容／材料想请你处理：……
我需要判断一个选择：……
这件事一直推进不下去：……
```

例如：

```text
/dbs 我做少儿编程课，已经有 40 个付费学员，但续费低。我不知道是产品问题、定价问题，还是我找错了人。
```

`/dbs` 会读当前对话里已经给出的信息，直接把你带到合适的 Skill。你无需重复描述背景。

## 怎样更新

已经安装 dbskill 后，直接对当前 Agent 说：

```text
更新 dbskill
```

Agent 会同步官方 dbskill。你不需要复制更新命令；它不会更新其他 Skill，也不会修改你的 dbskill 存档。

## 已经知道要做什么

熟悉 dbskill 后，可以跳过 `/dbs`，直接调用具体 Skill。命令后面直接写材料或问题：

```text
/dbs-diagnosis 我想卖一门给自由职业者的 AI 课，怎样判断这门生意是否成立？
/dbs-hook 这是我短视频的前 20 秒，帮我优化开头：……
/dbs-benchmark 我想研究做企业服务内容的账号，帮我找对标。
```

每个 Skill 都会继续追问真正缺少的信息。已经出现在当前对话里的信息，不需要重新回答。

## dbskill 怎样工作

1. 你提交当前的处境、目标、材料和卡点。
2. `/dbs` 判断当前最需要完成的任务，并调用一个 Skill。
3. Skill 直接产出判断、内容、方案、文件或行动。
4. 你补充反馈后，再由 `/dbs` 根据新结果判断下一步。

熟悉具体 Skill 后可以直接调用；第一次使用时，直接把任务交给 `/dbs` 即可。

#### `/dbs-skill-cleaner`｜本地 skill 清理器

**什么时候用：** 你担心本地 skill 含有广告导流、隐蔽商业意图、任务劫持、可疑外部调用或敏感数据读取，想先审查再处理。

**你可以这样说：** `/检查 skill`、`/清理 skill` 或「扫描本地所有 skill，看看有没有广告和越权行为」。

**你会得到：** 一份带文件位置、行号和风险理由的只读审查报告。对于需要处理的 skill，工具会在你确认具体路径后解除软链或移入本地隔离区，保留恢复能力。

**继续推进：** 完成本 Skill 后输入 `/dbs`。它会读取本轮结论和你的最新目标，再决定下一步。

## 常见场景与动态导航

dbskill 不预设固定执行链。每次只调用当前最合适的一个 Skill；完成后输入 `/dbs`，它会结合刚才的结论、你的补充和目标变化，重新判断下一步。

### 1. 从零开始做内容

```text
/dbs
```

先说清自己的业务、受众与目标。当前问题可能进入内容诊断、开头优化、标题生成或逐字稿检查；由当前结论决定。

### 2. 判断一个生意值不值得做

```text
/dbs-diagnosis
```

先拆清生意和问题。诊断后的结果可能需要寻找现实参照、沉淀长期决策，或先做一项验证动作。

### 3. 从爆款中提炼下一条内容

```text
/dbs-spread
```

先分析已有内容击中的心理与立场。下一步可能需要扩展讨论，也可能可以直接进入内容生产；由分析结论决定。

### 4. 写完一篇内容到发布

```text
/dbs-resonate
```

先检查内容是否真正击中人。结论会指出当前更优先处理共鸣、逻辑衔接、表达问题或发布排版。

### 5. 知道该做却一直做不动

```text
/dbs-action
```

先找行动卡住的具体信号。行动诊断会判断该审计目标、澄清业务问题，还是评估值得承担的摩擦。

### 6. 把本地文件夹变成知识库

```text
/dbs-knowledge
```

指定一个已有资料的文件夹，Agent 会先检查目录和版本关系，再建立可维护的知识库导航。完成后可以直接说「从知识库找一下……」「把这份资料放进知识库」或「检查一下知识库」。

### 7. 建立自己的内容资产与决策记忆

```text
/dbs-content-system
```

先把旧文稿、案例和课程变成结构化资产。后续行动根据素材规模、主题地图和当前业务任务重新判断。

## 使用习惯

- **材料越真实，结果越具体。** 说清楚你卖什么、面对谁、已经试过什么、哪里失败了。
- **不用重复背景。** `/dbs` 和各个 Skill 应先利用当前对话里的信息；只补充真正缺少的事实。
- **跨对话请存档。** 当前聊天记录关闭后，输入 `/dbs-restore` 拉回上次结论。
- **不知道下一步时回到 `/dbs`。** 它会按当前结论推荐值得继续走的方向。
- **一次只决定一步。** 当前结论和你的新反馈会改变下一次路由，因而不预先安排固定长链。
